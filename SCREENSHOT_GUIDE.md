# App Store Screenshot Guide for Artha

## Overview

Screenshots are critical for App Store conversion. They should:
1. Highlight what makes Artha unique vs Dime
2. Show Indian-specific features prominently
3. Be visually appealing and professional
4. Tell a story from left to right

## Technical Requirements

### Device Sizes Required

**iPhone** (6.5" Display - iPhone 14 Pro Max, 13 Pro Max, 12 Pro Max):
- Resolution: 1284 x 2778 pixels
- Required: Yes (most important)

**iPhone** (6.7" Display - iPhone 15 Pro Max, 15 Plus, 14 Plus):
- Resolution: 1290 x 2796 pixels
- Required: Yes if targeting latest devices

**iPhone** (5.5" Display - iPhone 8 Plus, 7 Plus, 6s Plus):
- Resolution: 1242 x 2208 pixels
- Optional but recommended for wider reach

**iPad Pro** (12.9" Display):
- Resolution: 2048 x 2732 pixels
- Required if iPad support is emphasized

### Format
- **File Type**: PNG or JPEG
- **Color Space**: sRGB or Display P3
- **Number**: 3-10 screenshots (recommend 6-8)
- **Order**: Left to right tells a story

## Screenshot Best Practices

### Design Tips

1. **Text Overlays**: Optional but recommended
   - Use semi-transparent backgrounds
   - Keep text short and punchy
   - Use emojis sparingly for personality
   - Ensure readability on both light/dark

2. **Sample Data**:
   - Use realistic Indian scenarios
   - Indian names: Rajesh, Priya, Amit, Sneha
   - Indian places: Bangalore, Mumbai, Delhi
   - Indian categories: Chai, Kirana, Auto
   - Realistic amounts: â‚¹50, â‚¹500, â‚¹5K, â‚¹50K

3. **Device Setup**:
   - Clean status bar (full battery, strong signal)
   - Time: 9:41 AM (Apple's standard)
   - Date: Current month
   - No notifications showing
   - WiFi enabled

4. **Visual Flow**:
   - Left to right tells a story
   - Start with "what is it"
   - Move to "unique features"
   - End with "why it's awesome"

## Testing Your Screenshots

Before final submission:

1. **View on Actual Device**: Transfer to iPhone, view in Photos
2. **Tiny Preview**: Screenshots look tiny in search results - are key features visible?
3. **Ask Others**: Show to 2-3 people unfamiliar with app - what do they understand?
4. **A/B Test** (after launch): App Store Connect allows testing different screenshots

## Iteration Plan

**Week 1 Launch**: Use basic screenshots (just app screens)
**Week 2-3**: Add text overlays based on user feedback
**Month 2**: Redesign based on which features users love most
**Month 3**: A/B test different screenshot orders
